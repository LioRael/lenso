# Lenso Product Blueprints V23 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build Product Blueprints & Addons on top of V22 Launchpad so users can list blueprints, create blueprint apps, add curated addons, diagnose local readiness, and see addon/doctor state in Console.

**Architecture:** Keep blueprints and addons as built-in CLI recipes, not a user-authored DSL. Reuse the existing host/service scaffold and existing JSON control-plane files. Host and Console stay read-only over generated state.

**Tech Stack:** Rust CLI, Rust admin-data endpoint, React Runtime Console, Markdown docs, JSON fixtures.

---

## File Structure

- `lenso-cli/src/main.rs`: add parser structs and dispatch for `app list`, `app inspect`, `app add`, and `dev doctor`.
- `lenso-cli/src/launchpad.rs`: hold blueprint/addon registry, app creation, addon application, doctor checks, and agent context additions.
- `lenso/crates/platform-admin-data/src/dto.rs`: add doctor response DTOs.
- `lenso/crates/platform-admin-data/src/handlers.rs`: add read-only `.lenso/dev-doctor.json` response helper and route handler.
- `lenso/crates/platform-admin-data/src/lib.rs`: register the doctor route.
- `lenso/contracts/openapi/app-api.v1.yaml`: generated by `just generate-contracts`.
- `lenso-runtime-console/src/data/available-modules.ts`: add doctor fetch/sample data.
- `lenso-runtime-console/src/pages/available-modules-model.ts`: add Launchpad addon and doctor response types.
- `lenso-runtime-console/src/pages/launchpad-model.ts`: summarize addons and doctor checks.
- `lenso-runtime-console/src/pages/launchpad-model.test.ts`: cover the new summaries.
- `lenso-runtime-console/src/pages/launchpad-page.tsx`: render addons and doctor state.
- `lenso-examples/fixtures/launchpad/support-desk-addon/`: generated addon and doctor fixtures.
- `lenso-site/content/docs/(host)/quickstart.mdx`: show `app add` and `dev doctor`.
- `lenso-site/content/docs/(host)/cli-reference.mdx`: document new commands.
- `lenso-site/content/docs/(host)/product-blueprints.mdx`: explain blueprints and addons.

Algorithm:

```rust
pub(crate) fn add_app_addon(options: AppAddOptions) -> Result<()> {
    let repo_root = Path::new(".");
    let mut state = read_launchpad_state_required(repo_root)?;
    let addon = addon_by_name(&options.addon)?;
    ensure_addon_supports_blueprint(&addon, &state.blueprint)?;
    ensure_addon_not_applied(&state, &addon.name)?;
    scaffold_missing_addon_services(&addon)?;
    upsert_system_addon(Path::new(SYSTEM_FILE), &addon)?;
    upsert_workspace_addon(Path::new(WORKSPACE_FILE), &addon)?;
    state.addons.push(launchpad_addon_from_addon(&addon));
    state.checklist.push(LaunchpadChecklistItem {
        id: format!("addon-{}", addon.name),
        label: format!("Addon {} configured", addon.label),
        status: "done".to_owned(),
        next_command: None,
    });
    write_json(Path::new(LAUNCHPAD_FILE), &state)?;
    println!("Added addon {}.", addon.name);
    println!("Next: lenso dev doctor");
    Ok(())
}
```

Use `serde_json::Value` helpers to upsert `services`, `modules`, and `dependencies` arrays by `name` or `from/to/capability`. Do not introduce typed system-manifest structs in V23.
