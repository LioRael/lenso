# Lenso App Change Plan V25 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build V25 App Change Plan so generated Launchpad apps can plan, review, apply, and display safe generated-state changes.

**Architecture:** Keep planning in `lenso-cli`; Host and Console only read `.lenso/app-change-plan.json`. Reuse V24 Launchpad/App Proof patterns and only write generated control-plane files plus missing service scaffolds. Do not add blueprint registries, signatures, or source migrations.

**Tech Stack:** Rust CLI, Rust admin-data endpoint, OpenAPI YAML, React Runtime Console, JSON fixtures, MDX docs.

---


- [ ] Add model types:

```rust
#[derive(Debug, Clone, Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
struct AppChangePlanState {
    protocol: String,
    status: String,
    generated_at_unix_ms: u64,
    project_name: Option<String>,
    blueprint: Option<String>,
    addons: Vec<String>,
    proof_status: Option<String>,
    changes: Vec<AppChangePlanItem>,
    blocked: Vec<AppChangePlanItem>,
    next_command: Option<String>,
}

#[derive(Debug, Clone, Deserialize, Serialize)]
#[serde(rename_all = "camelCase")]
struct AppChangePlanItem {
    id: String,
    kind: String,
    name: String,
    action: String,
    safe: bool,
    message: String,
    command: Option<String>,
}
```

- [ ] Implement `app_plan` by reading Launchpad, App Proof, workspace, and system state. It should produce `ready` when no changes are needed, `changes` when safe differences exist, `blocked` when unsafe or unsupported addon requests exist, `failed` for unreadable state, and `empty` for missing Launchpad.
- [ ] Implement `app_upgrade` as a thin alias around `app_plan`: `--check` exits non-zero for `changes`, `blocked`, or `failed`; `--write-plan` writes the plan.
- [ ] Implement `app_apply` by reading a plan, rejecting blocked/unknown/wrong-project plans, and applying only known safe change kinds. Reuse the V24 safe repair helpers where possible.
- [ ] Extend `agent_context_markdown` to include App Change Plan when `.lenso/app-change-plan.json` exists.
- [ ] Add unit tests for ready, changes, blocked unsupported addon, apply dry-run source boundary, and agent context change-plan text.
