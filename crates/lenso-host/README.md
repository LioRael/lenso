# lenso-host

`lenso-host` is the current host-facing API for Lenso applications generated by
`lenso host init`.

Use it from a host project as a Git-pinned dependency:

```toml
lenso-host = { git = "https://github.com/LioRael/lenso", rev = "<commit>", package = "lenso-host" }
```

The facade intentionally stays small:

- `HostBuilder`, `HostComposition`, and `HostLinkedModule` for host-owned linked modules;
- `run_api_from_env_with_composition`, `run_worker_from_env_with_composition`, and `run_migrations_from_env_with_composition`;
- `Migration` and `ModuleManifest` re-exports for linked module metadata;
- `lenso_host::http` helpers for linked Axum routes and OpenAPI registration.

Application SQL, repositories, auth/session policy, CRUD shape, and Runtime
Console UI stay in the host application or module code.

`lenso-host` is not crates.io-ready yet. It depends on Lenso boot crates that are
currently repository-internal, so generated hosts should pin a Git tag or commit
for reproducible builds.
